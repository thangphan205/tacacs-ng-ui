# tacacs-ng-ui — Frontend

React + TypeScript web interface for the [tacacs-ng-ui](../README.md) TACACS+ management platform.

## Tech Stack

| Layer | Library | Version |
|-------|---------|---------|
| Framework | React | 19 |
| Build | Vite + SWC | 8 |
| Language | TypeScript | 5 |
| UI Components | Chakra UI | 3 |
| Charts | Chakra UI Charts (Recharts) | 3 |
| Routing | TanStack Router (file-based) | 1 |
| Server State | TanStack React Query | 5 |
| Forms | React Hook Form | 7 |
| Linter/Formatter | Biome | 2 |
| E2E Tests | Playwright | 1.59 |

## Development Setup

### With Docker (recommended)

From the project root:

```bash
docker compose watch   # hot reload on every save
```

Frontend runs at **http://localhost:5173**, backend at **http://localhost:8000**.

### Local (without Docker)

```bash
cd frontend
npm install
npm run dev            # Vite dev server → http://localhost:5173
```

Requires the backend running separately (`fastapi dev app/main.py` in `backend/`).

## Available Scripts

```bash
npm run dev              # Vite dev server with HMR
npm run build            # TypeScript check + production build
npm run lint             # Biome format + lint (writes in place)
npm run preview          # Preview production build locally
npm run generate-client  # Regenerate TypeScript API client from OpenAPI schema
```

## API Client Generation

After any backend route or model change, regenerate the client from project root:

```bash
bash scripts/generate-client.sh
```

This exports the OpenAPI schema from the local backend venv and runs `npm run generate-client`. Requires `uv sync` completed in `backend/` first.

Commit the resulting changes in `frontend/src/client/`.

## Authentication

JWT token stored in `localStorage.access_token`. Configured in `main.tsx`. Auto-logouts on 401/403 responses. Supports:
- Email + password
- Google OAuth2
- Keycloak OIDC
- Passkeys (WebAuthn)

## Environment Variables

| Variable | Default | Description |
|----------|---------|-------------|
| `VITE_API_URL` | *(empty — uses same origin)* | Backend base URL for local dev against a remote API |

Set in `frontend/.env`:

```env
VITE_API_URL=http://localhost:8000
```

## End-to-End Testing

Requires Docker Compose stack running:

```bash
docker compose up -d --wait backend

npx playwright test           # headless
npx playwright test --ui      # interactive UI mode
```

Tear down and clean test data:

```bash
docker compose down -v
```

## Dark Mode

Implemented via `next-themes`. Toggled from the top navigation bar. Theme preference persisted in `localStorage`.
